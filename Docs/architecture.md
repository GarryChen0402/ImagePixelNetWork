# 网络架构设计

> **版本**: v0.1 | **日期**: 2026-05-24 | **所属项目**: [ImagePixelNetWork](./README.md)

## 整体架构

```
┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────────────┐
│  Encoder │ ──→ │Bottleneck│ ──→ │ Decoder  │ ──→ │ PaletteQuantizer │ ──→ 输出
└──────────┘     └──────────┘     └──────────┘     └──────────────────┘
```

## 1. Encoder（编码器）

采用 ResNet 风格的卷积编码器，逐步下采样提取特征。

| 层级 | 类型 | 通道数 | 下采样倍数 | 说明 |
|------|------|--------|------------|------|
| Conv1 | 7×7 Conv + BN + ReLU | 64 | 1× | 初始特征提取 |
| Block1 | 2× ResBlock | 64 | 1× | 浅层纹理 |
| Block2 | 2× ResBlock | 128 | 2× | 边缘、角点 |
| Block3 | 2× ResBlock | 256 | 4× | 形状、结构 |
| Block4 | 2× ResBlock | 512 | 8× | 语义特征 |

**设计考量**：下采样倍数不宜过大（最高 8×），因为像素风格仍需要保留一定的空间结构信息。

## 2. Bottleneck（瓶颈层）

- 6 个连续的 ResBlock (512 通道)
- 不进行额外的空间压缩
- 作用：在深层特征空间中学习从自然图像到像素艺术的内在映射关系

## 3. Decoder（解码器）

采用渐进式上采样 + 跳跃连接结构，融合 Encoder 的多尺度特征。

| 层级 | 类型 | 通道数 | 上采样 | 跳跃连接来源 |
|------|------|--------|--------|-------------|
| Up4 | Upsample + 2× ResBlock | 256 | 2× | Encoder Block3 |
| Up3 | Upsample + 2× ResBlock | 128 | 2× | Encoder Block2 |
| Up2 | Upsample + 2× ResBlock | 64 | 2× | Encoder Block1 |
| Up1 | 7×7 Conv → 3 (RGB) | 3 | — | — |

**上采样方式**：使用 PixelShuffle（亚像素卷积），而非简单的双线性插值，以获得更锐利的像素边缘。

**跳跃连接**：将 Encoder 对应尺度的特征与 Decoder 上采样结果进行通道拼接，帮助恢复空间细节。

## 4. PaletteQuantizer（可微分调色板量化层）

这是本项目的核心创新模块，负责将连续颜色映射到离散调色板。

### 设计思路

```
输入特征 (H, W, 3) 
    │
    ├──→ 计算与调色板每个颜色的 L2 距离
    │       │
    │       └──→ Softmax( -distance / τ )  获得软分配权重
    │                  │
    │                  ├──→ argmax → 硬分配（推理时）
    │                  └──→ softmax → 软分配（训练时）
    │
    └──→ 输出量化后的 RGB 图像 (H, W, 3)
```

### 关键参数

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| `palette_size` | 32 | 调色板颜色数量 |
| `temperature` τ | 1.0 → 0.1 | 退火参数，训练中逐步降低以逼近硬量化 |
| `palette_init` | k-means | 调色板初始化方式 |

### 温度退火策略

```
τ(t) = max(τ_min, τ_start × exp(-decay_rate × t))
```

- τ_start = 1.0
- τ_min = 0.1
- decay_rate = 1e-4

较高的 τ 使梯度平滑，利于早期训练；较低的 τ 产生更锐利的量化效果。

### 调色板设计

调色板本身是可学习的参数，训练过程中自动调整颜色以适应数据集风格。
对于 Minecraft 风格，可以在 Minecraft 贴图数据集上预聚类得到初始化调色板。

## 5. 损失函数

### 总损失

```
L_total = λ_content × L_content + λ_style × L_style + λ_tv × L_tv + λ_palette × L_palette
```

| 损失项 | 权重 | 说明 |
|--------|------|------|
| L_content | 1.0 | VGG 感知损失（L1 特征距离），保持内容结构 |
| L_style | 10.0 | Gram 矩阵风格损失，匹配像素艺术纹理 |
| L_tv | 1e-4 | 总变分损失，抑制高频噪声 |
| L_palette | 0.1 | 调色板正则化，惩罚颜色使用不均衡 |

### VGG 特征层选择

- 内容损失：VGG-19 的 `relu4_2` 层
- 风格损失：VGG-19 的 `relu1_1, relu2_1, relu3_1, relu4_1, relu5_1` 层

### 像素边缘增强

额外引入一个边缘感知损失：对输出图像计算 Laplacian 边缘图，惩罚模糊的过渡区域。
这有助于生成像素艺术特有的锐利边缘。

## 6. 推理时的 Grid 化后处理

为了增强 Minecraft 特有的"方块"感，推理时可对输出进行可选的后处理：

1. 将图像分割为 `16×16` 或 `8×8` 的 grid
2. 每个 grid 内的像素取众数颜色
3. 微调 grid 边界处的颜色一致性

这一步可作为可选的纯后处理，不参与训练，但在推理时显著提升"方块化"视觉效果。
